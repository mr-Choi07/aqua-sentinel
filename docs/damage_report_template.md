# 피해 상황 정리 보고서 — 항목 구조 및 매핑

## 출처

자연재난 구호 및 복구 비용 부담기준 등에 관한 규정 [별지 제1호서식]
자연재난 피해신고서 (law.go.kr, 개정 2018. 7. 24.)

이 서식은 **범용 자연재난 서식**(어업 전용 아님)이다. 우리 시스템은 이 서식의 항목
구조 중 어업 피해 신고 준비에 필요한 부분만 선별해서 참고용 문서를 만든다. 실제
신고는 관할 시·군·구/읍·면·동에 원본 서식으로 접수해야 하며, 이 문서는 그 준비를
돕는 참고 자료일 뿐 원 서식을 대체하지 않는다.

법령은 수시로 개정되므로, 실제 제출 전에는 law.go.kr에서 최신 시행본인지 다시
확인할 것.

## 1. 고정 안내문구 (PDF 최상단·최하단, 굵게)

> 본 문서는 실제 자연재난 피해신고서 양식이 아니며, 신고 전 참고자료입니다.
> 정식 신고는 관할 시·군·구/읍면동 사무소에 원본 서식으로 접수해야 합니다.

## 2. 어업인 정보 (원 서식 "1. 피해자 정보" 중 발췌)

원 서식에는 주거 형태, 가족 수, 고등학생 수, 재난지원금 지급계좌, 도시가스
사용여부 등 일반 재난(주택 피해 등)에 필요한 항목이 있으나, 어업 피해와 무관하고
불필요한 개인정보라 제외한다.

| 원 서식 항목 | 사용 여부 | 우리 앱 매핑 |
|---|---|---|
| 성명(대표자) | 사용 | 어업인명 |
| 주소(사업장) | 사용 | 어장 소재지 |
| 연락처 | 사용 | 연락처 |
| 주거 형태 | 제외 | - |
| 가족 수 / 고등학생 수 | 제외 | - |
| 재난지원금 지급통장 계좌번호 | 제외 | - |
| 도시가스 사용여부 | 제외 | - |

## 3. 피해 내용 (원 서식 "2. 피해 내용" 매핑)

원 서식 작성방법 3, 4항: "피해 구분란에는 담당 공무원이 현지 확인 후 매몰·침수·
소파·반파·전파·유실 중 해당하는 것을 적습니다", "음영처리된 부분은 담당 공무원이
현지 확인 후 적습니다." — 이 두 항목은 **시스템이 절대 채우지 않는다.**

| 원 서식 항목 | 우리 앱 매핑 | 채우는 주체 |
|---|---|---|
| 피해시설명 | 어장명 | 사용자 입력 |
| 총면적(소유+임차) | 양식 면적(ha) | 사용자 입력 |
| 면허·허가·등록번호 | 어업면허/신고번호 | 사용자 입력 (자동채움 금지) |
| 피해물량 - 신고 | AI 폐사 추정치 | **시스템 자동 기입** (참고용) |
| 피해물량 - 확정 | 공란, "공무원 현지확인 후 기재" 안내 | **공무원만 기재** |
| 피해 구분 (매몰/침수/소파/반파/전파/유실) | 공란, "공무원 현지확인 후 기재" 안내 | **공무원만 기재** |
| 피해 원인 | "AI 추정 원인(참고용, 공식 판정 아님)" 라벨 + 사진 분석 결과 | 시스템 자동 기입 |

## 4. 판단 근거는 출처별로 섹션 분리

관측 데이터(수온 기반 위험도)와 사진 기반 AI 소견은 서로 다른 신뢰도·성격의
정보이므로 문서에서 명확히 구분한다.

- **4-1. 관측 데이터 기반 판단** — `pipeline/risk.py`의 위험도 등급, 현재/72시간
  예상 수온, 판단 근거
- **4-2. 사진 기반 AI 소견** — `pipeline/damage_report.py`의 `analyze_damage_photo`
  결과 (원인 단정 금지, 전문가 확인 필요 문구 유지)

## 5. 불일치 경고 (휴리스틱)

관측 위험도가 "정상"인데 사진 소견 텍스트에 대량 폐사를 시사하는 표현이 감지되면
문서 상단에 경고를 표시한다.

> ⚠ 관측 수온과 사진 소견이 다릅니다. 즉시 신고를 권장합니다.

이 판단은 사진 분석 결과 텍스트에서 "대량", "다수", "폐사체" 등의 키워드를 찾는
단순 매칭이며, 정교한 판정이 아니다. 키워드가 없다고 해서 피해가 없다는 뜻은
아니고, 반대로 키워드가 있어도 실제로는 경미할 수 있다 — 참고용 신호일 뿐이다.
